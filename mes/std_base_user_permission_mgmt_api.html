1. 베이스 URL·환경

환경 동작
Vite dev + VITE_MES_DB_API_BASE 비어 있음 base === '' → /db/..., /func/... 상대 경로. renderer/vite.config.ts의 server.proxy로 https://100.71.84.10:7443에 전달.
프로덕션 기본 https://100.71.84.10:7443 + path.
절대 URL 예 https://100.71.84.10:7443/func/fn_user_authority_base?p1=MM

2. 엔드포인트 요약

구분 Method Path 용도
왼쪽 부서 콤보 GET /func/fn_cm_code p1=DEPT&p2=disp_seq
왼쪽 그리드(부서별 사용자) GET /func/fn_cm_user_dept p1=<dept>
왼쪽 아래(사용자 권한그룹) GET /func/fn_user_authority_userid p1=<user_id>
오른쪽 권한그룹 콤보 GET /func/fn_cm_code p1=USER_CLASS&p2=disp_seq
오른쪽 그리드(권한별 사용자) GET /func/fn_user_authority_base p1=<user_class>
저장 테이블 GET/POST/DELETE /db/tb_mes_user_authority 권한 매핑 CRUD

3. GET — /func/fn_cm_code?p1=DEPT&p2=disp_seq (왼쪽 부서 콤보)

3.1 호출 코드

  • 파일: fnCmCodeApi.ts — fetchFnCmCodeOptions('DEPT', 'disp_seq')

3.2 URL

GET {base}/func/fn_cm_code?p1=DEPT&p2=disp_seq

3.3 호출 시점·조건

시점 조건 트리거
화면 최초 마운트 항상 1회 useEffect — USER_CLASS와 Promise.all

3.4 응답·UI

  • { code, code_name } → { value, label }. UI에서 전체 옵션(value="")을 맨 위에 추가.
  • 부서 선택 시 왼쪽 그리드(fn_cm_user_dept) 자동 재조회.

3.5 에러

원인 동작
HTTP 오류 throw → 배너 콤보 로드 실패: …

8. POST / DELETE — /db/tb_mes_user_authority

8.1 테이블 구조

필드 비고
user_id PK 일부
user_class PK 일부 — USER_CLASS 코드 값
create_date
created_by
update_date null 가능
updated_by

8.2 POST (권한 추가)

POST {base}/db/tb_mes_user_authority
Content-Type: application/json
필드 출처
user_id 왼쪽 그리드 선택 사용자
user_class 오른쪽 콤보 선택 권한그룹 코드

8.3 DELETE (권한 제거)

DELETE {base}/db/tb_mes_user_authority?user_id={id}&user_class={cls}

10. 보완·주의점(제안)

  1. 데이터 247건: tb_mes_user_authority에 현재 247행, 18개 user_class에 분포.
  2. 복합 PK: user_id + user_class이므로 DELETE 시 두 값 모두 필수.
  3. 권한복사 탭: PNG에 "권한복사" 탭이 보이나 현재 구현은 조회 위주 — 권한 일괄 복사 기능 추가 시 별도 POST 로직 필요.
  4. fn_cm_user_dept 전체 조회: 부서 콤보에서 "전체" 선택 시 빈 p1을 보내면 전 부서 사용자를 반환하는지 백엔드 확인 필요.
