프로덕션(Electron 빌드 등) 기본 https://100.71.84.10:7443 + path, 또는 VITE_MES_DB_API_BASE.
절대 URL 예 https://100.71.84.10:7443/func/fn_unit_process_line?p1=

2. 엔드포인트 요약

구분 Method Path 용도

이 화면은 저장·삭제 HTTP 호출 없음(showFilterSave={false}). 그리드는 읽기 전용 표시.

3. GET — /func/fn_cm_code (p1=UNIT_PROCESS)

3.1 호출 코드

  • 파일·함수: fnCmCodeApi.ts — fetchFnCmCodeOptions('UNIT_PROCESS', 'disp_seq')
  • 화면: std_base_unit_process_line_mgmt.tsx — 마운트 시 useEffect 1회

3.2 URL·쿼리

GET {base}/func/fn_cm_code?p1=UNIT_PROCESS&p2=disp_seq
파라미터 전송 조건 의미
p1 항상 UNIT_PROCESS — tb_cm_code.code_group 등 서버 정의
p2 항상 disp_seq

3.3 호출 시점·조건

시점 인자/조건 트리거
최초 마운트 위 인자 useEffect (빈 deps)
이후 자동 재호출 없음 —

3.4 응답 처리

  • JSON 배열 또는 래핑 객체(data / rows / result / items / records / list 등) 중 첫 배열.
  • 행 { code, code_name } → { value, label } (rowToOption).
  • 실패 시 setUnitProcessOptions([]).

3.5 에러

원인 동작
res.ok === false throw → catch → setUnitProcessOptions([])

4. GET — /func/fn_unit_process_line

4.1 호출 코드

  • 파일·함수: fnUnitProcessLineApi.ts — fetchFnUnitProcessLine(p1)

4.2 URL·쿼리

전체(빈 p1):

GET {base}/func/fn_unit_process_line?p1=
  • 클라이언트는 먼저 위 URL을 호출하고, 파싱 후 행이 0건이면 쿼리 없이 GET {base}/func/fn_unit_process_line 를 한 번 더 시도한다.

단위공정 코드 조건(예: FCT):

GET {base}/func/fn_unit_process_line?p1=FCT
파라미터 전송 조건 의미
p1 항상(URLSearchParams) 조회줄 단위공정 PlainSelect 값. 빈 문자열 = 전체

4.3 호출 시점·조건

시점 인자/조건 트리거
최초 마운트 loadGrid('') useEffect — 전체 조회 1회
조회줄 Search loadGrid(unitProcessFilter) onFilterSearch
콤보만 변경 — 자동 조회 없음

4.4 응답 처리

  • JSON 배열 또는 래핑 객체 내 첫 배열; 객체 안 임의 키의 배열 값도 탐색.
  • 각 행 rowToGridRow → 6열 문자열 배열 (아래 §6).
  • 응답이 객체 배열이면 우선 필드명으로 매핑; 배열의 배열이면 인덱스 0~5로 자름.

4.5 에러

원인 동작
res.ok === false throw new Error(\GET fn_unit_process_line ${status}: …`)`
화면 setErrorMsg, setRawRows([])

7. 보완·주의점(제안)

  1. MesDataGridPanel: 데이터 열이 모두 빈 문자열이면 행이 렌더에서 제외됨 — API 키와 매핑 불일치 시 빈 그리드로 보일 수 있음.
  2. 전체 조회 0건 시 재시도: ?p1= 후 빈 배열이면 쿼리 없는 URL로 재호출 — 백엔드가 둘 중 하나만 지원할 수 있음.
  3. 배열 행 응답: 객체가 아닌 [[...],...] 이면 필드명 매핑 없이 열 순서만 사용.
  4. 저장 없음: 편집·POST 연동은 없음.
