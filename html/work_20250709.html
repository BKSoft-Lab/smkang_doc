2. 원인 분석
2.1 ART의 캐시 구조
Android Runtime(ART)은 앱 성능 최적화를 위해 DEX 파일을 OAT/VDEX 캐시로 변환하여
/data/dalvik-cache에 저장한다.
이 캐시는 다음 요소들과 연동되어 생성 및 검증된다:
DEX 파일 위치 (dex_file_location)
ClassLoaderContext 정보
bootclasspath 해시
ART compiler version 등
2.2 문제의 핵심 원인: 경로 mismatch
빌드 시 ART는 ClassLoaderContext에 다음 경로를 문자열로 기록:
/system_ext/framework/taurus.cl_prj.jar
실제 런타임에서는 symlink(/system_ext → /system/system_ext symlink) 가 풀리며 다음 경로로 해석됨:
/system/system_ext/framework/taurus.cl_prj.ja
r
ART는 경로가 문자 단위로 일치하지 않으면 캐시를 무효로 판단
→ expected != found → 캐시 로딩 거부
→ fallback 으로 APK 내부 classes.dex 로딩 시도
그러나 ClassLoaderContext가 여전히 꼬여 있어 fallback 도 실패 → ClassNotFoundException 발생
2.3 심볼릭 링크 삭제의 무의미함
/system_ext → /system/system_ext symlink 는 시스템 호환성을 위한 구조
이 symlink 를 삭제해도 ClassLoaderContext mismatch 문제는 전혀 해결되지 않음
오히려 부팅 불능, framework jar 탐색 실패 등 치명적 부작용 발생 가능

4. 해결 방안
4.1 빌드 플래그 수정
device.mk 또는 BoardConfig.mk에 아래 설정을 추가:
PRODUCT_DEX_PREOPT_BOOT_FLAGS := --classpath-dir=/system/system_ext/framework
해당 설정을 통해 빌드 시 symlink 풀린 경로를 ClassLoaderContext에 기록하도록 유도
이로써 ART의 문자열 비교에서도 mismatch 없이 통과
4.2 ART 캐시 삭제 및 재생성
빌드 플래그 적용 후 다음 명령으로 기존 캐시를 삭제:

rm -rf /data/dalvik-cache/*

rm -rf /data/misc/profiles/*

5. 심볼릭 링크 제거에 대한 판단
항목 내용
효과 mismatch 문제 해결되지 않음
위험성 시스템 부팅 실패, OTA 적용 실패 가능성 있음
결론 symlink 제거는 비권장. 빌드 설정으로만 해결 가능
6. OTA 적용과 ART 캐시 검증 절차
6.1 OTA 이후 동작 원리
OTA 적용 후 첫 부팅 시, ART는 기존 캐시가 유효한지 자동으로 검증한다:
OAT/VDEX 내부의 체크섬, 클래스 로더 경로, bootclasspath 등을 비교
일치하지 않으면 dex2oat를 통해 캐시를 다시 생성
6.2 캐시 무효화 발생 조건
조건 예시
시스템 이미지 변경 framework.jar, boot.oat 등 변경
APK 변경 system 앱 교체 등
symlink 구조 변경 /system_ext 경로 구조 변경
ART 버전 변경 컴파일러 옵션 변경 등
6.3 OTA 제작 시 주의사항
작업 권장 여부
정확한 classpath-dir 설정 ✅ 필수
ART 캐시 사전 생성 (dexpreopt) ✅ 권장
OTA 후 캐시 강제 삭제 ❌ 비권장 (부팅 시간 증가 우려)
7. 결론 및 권장사항
항목 설명
문제 핵심 ART ClassLoaderContext 경로 mismatch로 캐시 및 fallback 로딩 실패
최종 원인 symlink 경로와 실제 경로가 문자열로 달라서 ART가 캐시 무효로 판단
권장 해결책 빌드 플래그 수정 + 캐시 초기화로 정정된 context 기록 유도
피해야 할 방법 symlink 삭제, 비표준 경로 강제 변경 등은 시스템 위험 야기 가능
