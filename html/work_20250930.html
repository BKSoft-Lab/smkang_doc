UHD2가온 (외부기기 미연결) 스테레오 사운드 자동설정 및 무음현상

https://jira.kaonmedia.com/browse/K0644-391

단말 FW 2024112200에서 발생한 실시간TV 무음 증상에 대해 1차 로그 분석 결과

■ 증상 요약
구글 설정(System Audio + Dolby 5.1 활성 시) → TV에서 돌비로 인식, 케이블 볼륨 불가
기본 설정 강제 변경 시 → 모든 채널 무음 지속, OTT 서비스(Netflix, YouTube)는 정상 출력
강원지역 25년 7월 생산분 단말에서 총 7건 보고
■ 로그 분석 결과
nexus_audio_decoder 로그에서 “Unlicensed Algo (AC3 [UDC]) … Audio will be muted” 반복 출력
OTT 경로(AAC 기반)는 정상이나, 실시간 방송(AC3 포맷) 디코더가 라이선스 미적용 상태로 mute 처리
따라서 Dolby/AC3 라이선스 키 불일치/누락이 원인 가능성 높음
■ 정상 : System Audio + Dolby 5.1 활성 시
STB는 실시간 방송의 AC3 비트스트림을 그대로 TV로 패스스루(passthrough)
STB 내부 디코더를 거치지 않고 TV 쪽 Dolby 디코더가 직접 해석/출력 → TV가 라이선스 보유 상태라서 정상 음성 출력.
■ 비정상 : 기본 설정 (PCM 출력)
STB 내부 오디오 스택에서 AC3를 직접 디코딩 → PCM 변환 필요
STB의 nexus_audio_decoder가 AC3 라이선스를 인식하지 못해 Unlicensed Algo ... Audio will be muted 로 처리.
따라서 디코딩 단계에서 mute → 실시간TV 무음 발생.

9월29일 퀵으로 받는 2대 STB에 대한 디버깅 결과입니다.

■ 확인내용
공장초기화 및 펌웨어 재설치 시도 → 증상 복구 불가
nexus_audio_decoder: Unlicensed Algo (AC3 [UDC]) detected ... Audio will be muted 지속 발생
YouTube 및 DRM 기반 재생 → Widevine / PlayReady 키 정상 동작, 영상·음성 모두 정상
drm-info APK 확인 결과 : Widevine 및 PlayReady 라이선스 키 정상
ExoPlayer APK 테스트 : DRM 보호된 컨텐츠의 영상/음성 정상 재생
따라서 secure 영역 손상이나 SW 문제 가능성은 낮음
■ 판단
Widevine/PlayReady DRM 키는 정상 동작하므로 secure 영역은 정상으로 추정
무음 증상은 Dolby/AC3 방송 경로에서만 발생
생산 시 Dolby/AC3 라이선스 키 기록 과정의 누락 또는 오류 가능성이 가장 의심됨
■ 문의
생산팀
생산 공정 중 Dolby/AC3 키 주입 단계가 어떻게 관리되는지 확인 요청
특히, 생산 과정에서 AC3 음성 재생 여부를 실제로 검증(테스트 재생)하고 있는지 확인 필요
