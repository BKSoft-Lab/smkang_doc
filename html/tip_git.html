git diff 명령어로 생성한 패치 파일을 사용하여 코드를 현재 작업 디렉토리에 반영하는 방법

패치 파일 생성 (Optional): 패치 파일을 생성하려면 git diff를 사용하여 파일로 출력해야 합니다. 아래 명령어는 diff.patch라는 파일에 변경사항을 저장합니다.

git diff > diff.patch

패치 파일을 현재 작업 디렉토리에 반영: 이미 생성된 패치 파일을 적용하기 위해 git apply 명령어를 사용합니다.

git apply diff.patch

이 명령어는 diff.patch 파일에 기록된 변경 사항을 현재 디렉토리의 파일에 반영합니다.

변경 사항 확인: 변경 사항이 정상적으로 적용되었는지 확인하려면 git status 명령어로 확인할 수 있습니다.

git status

변경된 파일이 반영된 상태로 나와야 합니다.

수정 사항을 커밋 (Optional): 변경 사항을 커밋하려면 아래 명령어를 사용합니다.

git add .
git commit -m "Apply changes from diff.patch"

이렇게 하면 git diff로 만든 패치 파일을 현재 코드에 반영할 수 있습니다.
