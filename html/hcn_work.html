관련 정리

리모컨 특수 키
a. 히든메뉴 : 빨 5 + 노 1
b. 해상도 변경 : OK + 나가기 버튼 동시 10초
c. 시스템 정보 확인 빨 5 + 초 (*) 1
메뉴 5 → 4 바커채널 설정 ( 초기부팅 채널 )
메뉴 7 → 2 리모컨 수동 업데이트
d. BT 페어링/언페어링
페어링 : 홈+ OK 5초 이상
언페어링 : 홈 + 빨 5초이상

a. default 2-way ( Docsis 로부터 받음 )
b. 1-way( RF Overlay ) Docsis 옆에 LAN선으로 network 설정인 경우

조절된 적용 값은 cat /proc/brcm/audio_volume 정보로 확인 가능
c. youtueb, apk 에서 사용되는 볼륨 전체를 조절 해달라고 한다면, frameworks/av/services/audiopolicy/config/default_volume_tables.xml 수정

git log -p frameworks/base/services/core/java/com/android/server/audio/AudioService.java 참조

브로드컴 드라이버 로그 활성화 A ( 부팅 이후 특정 동작 확인시 로그 봐야 할 때 )
userdebug mode, 추가 빌드 필요 하지 않음. serial/shell root 권한. 모듈 별로 로그 설정 함.
예) qam 관련 로그 추가 시
echo "bads_docsis msg" > /proc/brcm/debug
echo "nexus_docsis_priv msg" > /proc/brcm/debug
echo 'nexus_docsis msg' > /proc/brcm/debug
cat /proc/brcm/debug | egrep -i 'docsis' << 여기서 'wrn' 이 'msg'로 바뀐것 확인가능
logcat -s nx-klog:*
브로드컴 드라이버 로그 활성화 B ( 부팅 후 바로 나오는 로그 확인 해야 할 때 )
코드에 삽입, 빌드 해야 함.
