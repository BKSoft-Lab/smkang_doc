알티 문의

문제 : 부팅초기 히든메뉴에서 ‘공장초기화’를 실행하면 sys.powerctl이 reboot,(인자 없음) 으로 호출되어 리커버리 진입 없이 그냥 재부팅 됩니다.
정상 동작: 정상 경로의 공장초기화는 sys.powerctl이 reboot,recovery 로 설정되어 Recovery 모드로 재부팅 → wipe 동작 수행합니다.
근거 로그
비정상(히든메뉴 초기 부팅 시):
getprop sys.powerctl → reboot,
reboot: Restarting system with command ''
정상(일반 경로):
getprop sys.powerctl → reboot,recovery
reboot: Restarting system with command 'recovery'

공장초기화 호출 방식
현재 히든메뉴에서 어떤 API/명령으로 호출 중인지?
android.os.RecoverySystem.rebootWipeUserData(...) 사용 여부
또는 PowerManager.reboot("recovery") / setprop sys.powerctl "reboot,recovery" / 단순 reboot 호출인지

효 과
수정 루틴이 “메인 플로우”로 동작해서 항상 리커버리에서 wipe를 보장함.
기존 루틴(MASTER_CLEAR + pm.reboot(null)) 은 그대로 존재해도 중복 무해이며, (알티미디어 수정 없이 동작)
